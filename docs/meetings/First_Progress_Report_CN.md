# 树状 RAG 与多步回溯导航 — 阶段工作说明

简要说明：**研究什么问题、核心创新与研究取向、系统如何工作、目前已做到哪一步**。

---

## 摘要

长文档问答若仅用扁平检索，难以同时利用 **主题层级** 与 **句级证据**；多步决策又容易 **早走偏支**。本工作以 **树状结构上的多步导航与回溯** 为**可检验场景**，核心科学关切是：**Mamba 固定维隐状态与节点级快照恢复，能否作为低开销、可恢复的路径状态载体**；**Router** 与 **SSGS（State-Snapshot Guided Search）控制器** 提供子节点排序、预算、接受与回溯等**显式控制**，用于**验证该载体**。生成侧采用成熟大模型，**只读整理后的文本证据**，便于把 **证据获取过程** 与 **答案生成** 在分析上分开。须正视 **路径混合** 等机制风险，并以消融与对照回答。计划先在树状 RAG 上收敛证据链，再审慎讨论图场景扩展。

---

## 1. 要解决什么问题


| 方面        | 内容                                                                                      |
| --------- | --------------------------------------------------------------------------------------- |
| **结构**    | 知识以 **树** （图）组织，兼顾高层概览与底层叶子，比纯扁平切块更贴近长文档与多跳阅读。                                          |
| **控制**    | 多步探索需要 **试探—评估—回溯**；需要明确 **何时扩展、何时回退、何时停止、证据如何进入上下文**。                                  |
| **状态与代价** | 长链路上若依赖随序列增长的中间状态， **回退与重算** 代价高；希望在导航侧探索 **固定大小状态 + 可恢复快照** ，探索有利于 **深层、可回溯** 树上的受控搜索。 |
| **分析需求**  | 需要能区分 **「从未走到含金证据的枝」** 与 **「走到了但未进入可用上下文」** 等，避免把生成端噪声误记为导航收益。                          |

## 2. 核心研究问题

树导航与回溯是**平台与手段**，**Mamba 快照载体**是**核心可检验对象**。


| 编号             | 问题                                                                                                                                   |
| -------------- | ------------------------------------------------------------------------------------------------------------------------------------ |
| **RQ1（机制与风险）** | 快照恢复后改走新路径时，**路径混合** 是否导致中间指标或表示的**不可控退化**？须用 **快照 vs 重编码前缀 vs 无回溯** 等消融回答。                                                          |
| **RQ2（载体与代价）** | 相对其他回溯信息载体，**固定维快照 + 恢复** 的 trade-off；**建议**加 **Transformer 编码 + 同等快照语义** 作中间基线，以分离「固定维快照形态」与「Mamba 选择性机制」。                          |
| **RQ3（条件与效力）** | 按 **树深度/扇出** 等分层的边际收益；**论文终稿**得含 **有限规模、协议预先声明** 的 **外部对照**（树搜索式多步 RAG、层次化页级/索引式导航等**方法族**）。**是否已跑完**以实验记录台账为准，**不在此条用「计划项」代替实验事实**。 |


另：**可重复、可度量的闭环与过程归因** 为工程与评测要求。

## 3. 核心创新与研究取向

**创新重心**：**Mamba 固定维隐状态 + 节点级快照恢复** 作为树上多步导航中的 **路径状态载体**；**树** 与 **显式回溯** 用于**提供可检验场景与协议**。

1. **机制层**：沿路径读树时以 Mamba 更新 **固定维隐状态**；在决策点 **保存/恢复快照**，使回溯在实现上**有明确载体**；**路径混合** 的风险以消融与讨论**正面处理**。
2. **控制层**：**SSGS** 与 **Router** 职责分离（排序与编码、决策边界清晰）；默认协议下 **Mamba 每步只编码「控制器选中的当前节点」**，Router 通过 **访问次序与剪枝** 间接塑造状态轨迹（见框架 **§6.2.1**）。当前以**规则控制**为主；**若资源允许**在主文或附录纳入**至少一个学习化变体**，否则须在局限中**明确交代**与强学习树策略未对齐的原因。
3. **取向层**：**先**建立可度量、可对照的 **导航—证据—答案** 归因链；**不**把「Mamba 做生成」或「无归因榜单跃升」写为主结论。
4. **延展**：图按查询展开为搜索树等，仅在树状主线与载体结论**站稳**后再扩展。

## 5. 从问题到答案：数据与推理流程

以下按 **单条问答样本** 说明 **信息如何流经各模块**。

1. **输入**
  系统获得 **自然语言问题** 及与之关联的 **树**（节点文本、父子关系、叶子与主题的对应关系等）。
2. **搜索控制器驱动多步导航**
  从根或当前焦点出发，在 **深度、节点数、证据数量** 等限制下，反复进行 **扩展子节点 → 评估当前路径与证据 → 必要时回溯**。  
   需要 **子节点排序** 时，控制器调用 **路由器**；需要 **把当前访问节点纳入可携带的路径表示** 时，调用 **Mamba 导航编码器**。
3. **导航编码器的一步**
  对 **当前节点文本** 做顺序编码，更新 **固定大小的隐状态**；在预定位置 **保存快照**；若控制器决定回溯，则 **恢复** 到先前快照并改试其他分支。其输出是 **供排序与控制使用的表示**，**不是**最终自然语言答案。**默认协议**：每步通常只编码 **本步被选中的一个节点**，而非把「全体兄弟按排序拼成一条再喂入」；若采用后者等变体，须在实验配置中**单独声明**（与框架 **§6.2.1** 一致）。
4. **路由器的一步**
  在候选子节点上，根据 **问题、子节点文本、导航器给出的特征** 等，按所选策略输出 **排序或分数**，交回控制器决定访问次序与剪枝。因此 Router **不直接改写单步 token 串**，但通过 **次序与剪枝** 影响 Mamba 在时间轴上见到的节点序列；做 **Router 消融** 时须考虑与 **载体/快照** 消融的 **交叉设计**。
5. **形成生成阶段可用的材料**
  导航结束后，得到 **证据文本列表** 及可选的 **路径摘要**；再经 **上下文选择与截断**（课题中另有独立约定），形成送入生成模型的 **可读上下文**。
6. **生成与评价**
  生成模型基于 **问题与证据文本** 输出答案。评价时可将 **「金证据是否曾被访问、是否进入上下文」** 等 **过程侧** 与 **答案正确率** 等 **结果侧** **分开讨论**，避免混因。

## 6. 目前已开展的工作

在 **真实公开数据子集** 上，已完成或正在推进的工作大致包括：

- **端到端链路**：从树与问题出发，经导航得到证据，再交由固定大模型生成答案的 **全流程可运行**；  
- **批量与可复现**：在统一样本清单上组织 **导航与可选端到端** 的批量运行，并保留 **批次与样本级记录**，便于复查与对比；  
- **过程度量**：引入 **接受门与审计类摘要**，用于区分 **「从未走到含金叶」** 与 **「已 visit 但未进入可接受证据」** 等，支撑 **归因与 RQ1/RQ2 实验设计**；  
- **策略探索**：在 **同一树与同一清单** 下，对比 **规则排序、余弦类排序、根层轻量可学习排序与规则混合** 等 **路由器变体**，并结合 **实体先验、探索预算** 等 **控制器侧** 旋钮做 **单变量** 与 **阶段性默认工作点** 收敛；  
- **叙事与判停规范**：约定 **过程指标与答案指标并列**、**无效批次不参与优劣比较** 等，避免过度解读单次波动。

具体 **配置、批次标识与数字表格** 需要量化材料时再另行提供。

## 7. 局限性与开放点

- **不能主张**：Mamba **全面优于** Transformer；在无严格归因下宣称 **榜单式全面领先**。  
- **Oracle**：仅作 **差距参照**，不与真实导航策略混为同一结论层次。  
- **路径混合与固定维容量**：选择性 SSM 下状态与路径耦合；**须**以消融与论文讨论**主动交代**（见框架 **§5.3**）。  
- **外部基线与树质量**：论文须含 **§4 RQ3** 所述 **方法族级** 外部对照与 **树接口假设**；**工程进度**与 **成稿主张** 分开核对，避免口头「计划」与表格不一致。  
- **Router—Mamba 协议**：默认 **松耦合于单步输入、强相关于轨迹**；协议变体不可与默认混表（见 **§6.2.1**）。  
- **RQ2 中间基线**：**Transformer + 快照** 若缺失，易把「固定维」与「Mamba」混为一谈。
