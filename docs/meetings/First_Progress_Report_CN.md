# 树状 RAG 与多步回溯导航 — 阶段工作说明

简要说明：**研究什么问题、核心创新与研究取向、系统如何工作、目前已做到哪一步**。

---

## 摘要

长文档问答若仅用扁平检索，难以同时利用 **主题层级** 与 **句级证据**；多步决策又容易 **早走偏支**。本工作以 **树状结构上的多步导航与回溯** 为**可检验场景**，推进上采用 **主张分层、协议分叉、叙事可合成**：

- **路线 A（工程—科学核心）**：**固定维快照回溯**、**SSGS / Router 显式控制** 与 **接受门、过程归因** 构成可复现闭环；在 **P0 诊断协议**（每步以**当前节点**为主输入，跨步路径信息以**向量级融合**与快照栈组织；与 **`SSGS_Research_Framework_CN.md` §6.2.1** 默认一致）下，**导航编码器可替换**——同一控制器与融合规则下可用句向量等基线对照（见 **`Navigation_Experiment_Record_CN.md`**「句向量诊断」），结论域写成 **「载体 + 协议 A」下的可替换性**，而非「无协议约束下任意编码器等价」。
- **路线 B（研究增量）**：**SSM 协议设计**——**不**把 Mamba 写成万能节点编码器；在拟议的 **P1 路径递归协议**（须 **独立 `batch_id` / 配置前缀与文档小节**，与 P0 **禁止混表**）中，将 Mamba 定位为 **可恢复路径状态载体**，以 **结构化路径输入** 与 **快照语义和访问链对齐**，使 **递归压缩与选择性扫描** 成为可消融对象；**证据原文**仍由读侧与生成侧承担，与 COPY/检索型文献短板**主动划界**。
- **合成叙事**：**Mamba 不是「多跳 QA 自动变聪明」的黑箱，而是树导航中在声明协议下可检验的「路径级状态载体」**；路线 A 提供尺子与平台，路线 B 在**分列协议**下追问架构优势是否被**打开**。

须正视 **路径混合** 等机制风险，并以消融与对照回答。计划先在树状 RAG 上收敛证据链与台账，再审慎讨论图场景扩展。

## 1. 要解决什么问题


| 方面        | 内容                                                                                      |
| --------- | --------------------------------------------------------------------------------------- |
| **结构**    | 知识以 **树** （图）组织，兼顾高层概览与底层叶子，比纯扁平切块更贴近长文档与多跳阅读。                                          |
| **控制**    | 多步探索需要 **试探—评估—回溯**；需要明确 **何时扩展、何时回退、何时停止、证据如何进入上下文**。                                  |
| **状态与代价** | 长链路上若依赖随序列增长的中间状态， **回退与重算** 代价高；希望在导航侧探索 **固定大小状态 + 可恢复快照** ，探索有利于 **深层、可回溯** 树上的受控搜索。 |
| **分析需求**  | 需要能区分 **「从未走到含金证据的枝」** 与 **「走到了但未进入可用上下文」** 等，避免把生成端噪声误记为导航收益。                          |

## 2. 核心研究问题

树导航与回溯是**平台与手段**；**固定维快照载体** 是 **路线 A** 的主线交付物，**Mamba/SSM 在扩展协议下的行为** 是 **路线 B** 的增量对象——二者 **可合成叙述**，但 **实验与表格须按协议分叉**（见 **`SSGS_Research_Framework_CN.md` §5.0**）。


| 编号             | 问题                                                                                                                                   |
| -------------- | ------------------------------------------------------------------------------------------------------------------------------------ |
| **RQ1（机制与风险）** | 快照恢复后改走新路径时，**路径混合** 是否导致中间指标或表示的**不可控退化**？须用 **快照 vs 重编码前缀 vs 无回溯** 等消融回答。                                                          |
| **RQ2（载体与代价）** | 相对其他回溯信息载体，**固定维快照 + 恢复** 的 trade-off；**建议**加 **Transformer 编码 + 同等快照语义** 作中间基线，以分离「固定维快照形态」与「Mamba 选择性机制」。                          |
| **RQ3（条件与效力）** | 按 **树深度/扇出** 等分层的边际收益；**论文终稿**得含 **有限规模、协议预先声明** 的 **外部对照**（树搜索式多步 RAG、层次化页级/索引式导航等**方法族**）。**是否已跑完**以实验记录台账为准，**不在此条用「计划项」代替实验事实**。 |


另：**可重复、可度量的闭环与过程归因** 为工程与评测要求。

## 3. 核心创新与研究取向（路线 A + 路线 B）

### 3.1 路线 A：固定维快照回溯作为核心工程—科学主张（P0 下编码器可替换）

**创新重心（第一梯队）**：在受控树上实现 **固定维状态 + 决策点快照保存/恢复**、**低成本回溯**，并与 **接受门、`accept_gate_audit`、分桶与 `event_log` 归因** 绑成 **同一套可复现合同**。在此意义上，**「快照—回溯—归因」** 是本工作**已落地、可写进贡献句的主干**。

在 **P0 诊断协议** 下，导航编码器的主要可观测行为接近 **「当前节点 → 向量；跨步历史 → 融合向量 + 栈」**；因此 **句向量等替代编码器** 与 Mamba **可对齐对照**（见实验记录 **「句向量诊断」**）。该结果支持 **路线 A** 的结论域：**在协议 A 下编码器可替换**；**不**被误读为「Mamba 无价值」，而应读作 **「优势通道未在 P0 中单独打开」**——从而自然衔接 **路线 B**。

### 3.2 路线 B：SSM 协议设计作为核心研究增量（激活架构优势）

**创新重心（第二梯队，须独立协议与批次）**：在 **P1 路径递归协议** 中显式引入 **路径条件化结构化输入**（首版：**`[Q]`/`[PATH]`/`[NODE]`** 单次 HF 前向，见 **`navigator_path_recursive_prompt`** 与 **`Navigation_Experiment_Record_CN.md`**「P1 路径递归协议」段）、使 **隐状态更新语义** 与 **树上访问链** 对齐；**分支切换标记 / 快照语义重整** 等仍属 **消融扩展**。**路径混合**（RQ1）、**与 Transformer+快照的公平对照**（RQ2）写入消融矩阵。**P1 与 P0 不可混表**：论文与台账须用 **不同小节名 + `batch_id` 前缀** 绑定。

**分工边界（与文献一致）**：Mamba/SSM **不承担**「原文级精确仓库」；**⑥ 生成** 只消费 **已定稿文本证据**，导航侧输出用于 **路由与过程判据**。

### 3.3 合成陈述（对外一句话）

**我们不是把 Mamba 当万能编码器，而是作为树导航中、在声明协议下的「可恢复路径状态载体」：路线 A 提供尺子与平台；路线 B 追问在结构化路径输入与快照语义对齐后，固定维递归与选择性是否带来可测增益。**

### 3.4 控制层、取向与延展

1. **控制层**：**SSGS** 与 **Router** 职责分离（排序与状态侧、决策边界清晰）；**P0** 下 Mamba 每步主要编码 **控制器选中的当前节点**，Router 通过 **访问次序与剪枝** 塑造时间轴上的节点序列（框架 **§6.2.1**）。**P1** 若改为路径拼接或批量兄弟编码等，须 **单独声明** 并与 P0 **分列对照**。
2. **取向层**：**先**建立可度量、可对照的 **导航—证据—答案** 归因链；**不**把「Mamba 做生成」或「无归因榜单跃升」写为主结论。
3. **延展**：图按查询展开为搜索树等，仅在树状主线与 **路线 A 台账** **站稳**、**路线 B** 有 **smoke→满量** 节奏后再扩展。当前以**规则控制**为主；**若资源允许**在主文或附录纳入**至少一个学习化变体**，否则须在局限中**明确交代**与强学习树策略未对齐的原因。

## 5. 从问题到答案：数据与推理流程

以下按 **单条问答样本** 说明 **信息如何流经各模块**。

1. **输入**
  系统获得 **自然语言问题** 及与之关联的 **树**（节点文本、父子关系、叶子与主题的对应关系等）。
2. **搜索控制器驱动多步导航**
  从根或当前焦点出发，在 **深度、节点数、证据数量** 等限制下，反复进行 **扩展子节点 → 评估当前路径与证据 → 必要时回溯**。  
   需要 **子节点排序** 时，控制器调用 **路由器**；需要 **把当前访问节点纳入可携带的路径表示** 时，调用 **Mamba 导航编码器**。
3. **导航编码器的一步（须标明 P0 / P1）**
  **P0（当前台账主协议）**：对 **当前节点文本** 做顺序编码，更新 **固定大小的隐状态**；在预定位置 **保存快照**；若控制器决定回溯，则 **恢复** 到先前快照并改试其他分支。其输出是 **供排序与控制使用的表示**，**不是**最终自然语言答案。跨步「路径感」主要来自 **向量级路径融合** 与 **快照栈**，**不等价于**将整段路径原文作为单次长序列送入 SSM 的递推（与框架 **§5.3** 一致）。**默认**：每步通常只编码 **本步被选中的一个节点**；若采用「全体兄弟拼接再编码」等变体，须在实验配置中**单独声明**（框架 **§6.2.1**），且与 P0 **分列 `batch_id`**。
  **P1（路径递归协议，计划中）**：在 **单独命名** 的配置下，引入 **路径条件化结构化输入** 与（若实现）**与分支切换对齐的快照语义**，使 SSM 的递归与选择性成为 **可对照变量**；上线前须经过 **小样本 smoke**，再决定是否纳入满 manifest 主表。
4. **路由器的一步**
  在候选子节点上，根据 **问题、子节点文本、导航器给出的特征** 等，按所选策略输出 **排序或分数**，交回控制器决定访问次序与剪枝。因此 Router **不直接改写单步 token 串**，但通过 **次序与剪枝** 影响 Mamba 在时间轴上见到的节点序列；做 **Router 消融** 时须考虑与 **载体/快照** 消融的 **交叉设计**。
5. **形成生成阶段可用的材料**
  导航结束后，得到 **证据文本列表** 及可选的 **路径摘要**；再经 **上下文选择与截断**（课题中另有独立约定），形成送入生成模型的 **可读上下文**。
6. **生成与评价**
  生成模型基于 **问题与证据文本** 输出答案。评价时可将 **「金证据是否曾被访问、是否进入上下文」** 等 **过程侧** 与 **答案正确率** 等 **结果侧** **分开讨论**，避免混因。

## 7. 局限性与开放点

- **不能主张**：Mamba **全面优于** Transformer；在无严格归因下宣称 **榜单式全面领先**；在 **P0 诊断结果** 上直接推出 **「P1 下 SSM 必然更强」**（两条协议须 **分列证据**）。  
- **Oracle**：仅作 **差距参照**，不与真实导航策略混为同一结论层次。  
- **路径混合与固定维容量**：选择性 SSM 下状态与路径耦合；**须**以消融与论文讨论**主动交代**（见框架 **§5.3**）。  
- **COPY/精确检索型短板**：固定维 SSM 在文献中有已知边界；本工作将 **路径级压缩与判据** 与 **原文证据** 分工，须在成稿中 **写清边界**，避免审稿人按「万能检索器」误读。  
- **外部基线与树质量**：论文须含 **§4 RQ3** 所述 **方法族级** 外部对照与 **树接口假设**；**工程进度**与 **成稿主张** 分开核对，避免口头「计划」与表格不一致。  
- **Router—编码器协议与 P0/P1 分叉**：默认 **松耦合于单步输入、强相关于轨迹**；**P0、P1 及 §6.2.1 变体** 均 **不可混表**。  
- **RQ2 中间基线**：**Transformer + 同等快照语义** 若缺失，易把「固定维形态」与「Mamba 选择性」混为一谈；**P1** 开跑后尤须预先设计 **公平回溯成本模型** 下的对照。
