# 树状 RAG 与多步回溯导航 — 阶段工作说明

简要说明：**研究什么问题、核心创新与研究取向、系统如何工作、目前已做到哪一步**。

---

## 摘要

长文档问答若仅用扁平检索，难以同时利用 **主题层级** 与 **句级证据**；多步决策又容易 **早走偏支**。本工作以 **树状结构上的多步导航与回溯** 为可检验场景，围绕两条核心问题展开：（1）**固定维快照回溯是否可行**——在深度与节点预算下能否稳定运行，并用过程指标区分「未触达金叶」与「已触达但未进入可用证据」等；（2）**能否用 SSM（Mamba）的递归状态承载路径相关信息，并与 Router 协同取得可测效果**——导航编码器更新隐状态，Router 对子节点排序，**SSGS**（State-Snapshot Guided Search）控制器统一裁决探索、接受与回溯；生成侧采用成熟大模型，**只读整理后的证据文本**，便于把 **证据获取** 与 **答案生成** 在分析上分开。

**数据**：当前主线实验基于 **2WikiMultiHopQA** 导出的真实子集与树结构。

**方法**：快照恢复后的路径混合风险通过消融检验；固定维状态的容量边界结合树结构变化（深度、扇出等）评估。

**问题立场（构思层）**：承认 **固定维状态** 在 **长轨迹与回溯** 下存在 **容量与路径依赖** 张力；论文叙事优先 **用协议与对照把「在何种契约下有用」写清楚**，而不是在缺乏实验处仅以辩论否认瓶颈。后续改进走 **「改协议 / 补对照 / 必要时引入训练目标」** 的分支，各分支须 **单独论证** 是否仍使 **SSM 在声明输入契约下具有不可替代性或明确优势**。

## 1. 要解决什么问题

| 方面 | 内容 |
| :--- | :--- |
| **结构** | 知识以 **树** 组织，兼顾高层概览与底层叶子，比纯扁平切块更贴近长文档与多跳阅读。 |
| **控制** | 多步探索需要 **试探—评估—回溯**；需明确 **何时扩展、何时回退、何时停止、证据如何进入上下文**。 |
| **状态与代价** | 若回溯依赖随序列膨胀的中间表示，**回退与重算**代价高；希望在导航侧探索 **固定大小状态 + 可恢复快照**，支撑 **深层、可回溯** 的受控搜索。 |
| **分析需求** | 区分 **「从未走到含金证据的枝」** 与 **「走到了但未进入可用上下文」** 等，避免把生成端噪声误记为导航收益。 |

## 2. 核心研究问题

树导航与回溯是 **平台与手段**；下面两条是 **并列的核心科学问题**，下文 RQ1—RQ3 为其分解与可检验子项。

### 问题一：固定维快照回溯是否可行

在树上多步搜索中，**决策点保存隐状态快照、回溯时恢复** 是否 **工程可行**（不崩溃、可复现），以及 **路径混合**（恢复快照后改走新分支）是否带来 **不可控的表示退化**。须通过 **快照 vs 重编码前缀 vs 无回溯** 等消融与过程指标回答。

**子问题（对应可写进论文的分解）**

| 编号 | 内容 |
| :--- | :--- |
| **RQ1** | 快照恢复后改走新路径时，**路径混合** 是否导致导航相关中间指标的显著退化？ |
| **RQ2** | 相对其他回溯信息载体，**固定维快照 + 恢复** 的代价—质量权衡；建议以 **同等快照语义下的 Transformer 类编码** 等为对照，分离「**快照形态**」与「**Mamba/SSM 选择性机制**」。 |

### 问题二：能否用 SSM 递归状态承载路径信息，并借助 Router 取得实际效果

一方面，在 **默认协议** 下，导航器对 **当前访问节点** 做编码、更新 **固定维隐状态**，跨步路径信息通过 **向量级融合与快照栈** 与控制器轨迹结合。目前 **Router** 在排序时使用 **问题与子节点文本** 及导航器给出的 **标量先验**（如节点级相关性），**不直接使用高维隐状态向量**。因此，在默认协议下，SSM 递归状态对路由决策的影响主要通过 **控制器的接受/拒绝等环节** 间接传导，而非通过 Router 的排序公式直接传导；即使 Navigator 内部维护路径感知的隐状态，其对子节点选择的影响仍受 **特征合同** 约束。

另一方面，计划考察 **扩展协议**：将 **路径条件化文本**（如问题 + 祖先摘要链 + 当前节点）并入 **单次前向**，使递归语义与访问链更对齐。

**子问题**

| 编号 | 内容 |
| :--- | :--- |
| **RQ3** | 在需要回溯的困难样本（筛选标准见实验记录）上，与无回溯基线及外部方法族（如 IRCoT 式迭代检索、NaviRAG 式层次导航）做 **有限规模** 比较；同时考察树深度、扇出变化对快照回溯效率的影响。论文成稿须 **预先声明协议与数据规模**，并包含方法族级外部对照（如树搜索式多步 RAG、层次化导航等）中的有限规模比较。 |

**工程要求**：可重复运行、可度量、可对过程做归因（与 RQ 配套）。

## 3. 核心创新与研究取向

1. **快照与载体**：以 **固定维隐状态 + 节点级快照恢复** 作为树上多步导航中 **可恢复的路径状态载体**；回溯与搜索协议提供 **可检验场景**。
2. **控制与协同**：**SSGS** 负责探索、预算、接受与回溯；**Router** 负责子节点排序。二者与导航编码 **职责分离**，由控制器在 **状态侧信号** 与 **排序侧信号** 汇合后统一决策。
3. **默认与扩展**：默认实现强调 **每步编码当前节点** 与 **可审计轨迹**；路径信息的更强注入可通过 **协议扩展** 与 **对照实验** 检验，避免与默认结论混写。
4. **取向**：优先建立 **导航—证据—答案** 的可对照归因链；**不以**「Mamba 替代大模型做生成」或「无过程依据的榜单跃升」为主结论。
5. **延展**：图按查询展开为搜索树等，在树状主线结论稳定后再讨论。

## 4. 系统构成与模块关系

下列为 **概念层级**；不涉及具体类名与文件名。下图整幅宽约 **33** 个等宽字符；请用 **Courier New / Consolas** 查看；复制到 Word 时请将该段设为等宽字体。

```
┌───────────────────────────────┐
│① 输入：问题 + 树（文档/图）        │
└───────────────┬───────────────┘
                │
                ▼
┌───────────────────────────────┐
│② SSGS 控制器                   │
│  探索·预算·接受·回溯            │
└───────┬───────────────┬───────┘
        │               │
        ▼               ▼
┌─────────────┐   ┌─────────────┐
│③ Mamba      │   │④ Router     │
│ 导航编码     │   │ 子节点排序   │
│ 隐状态·快照  │   │ 规则/余弦/学习│
└───────┬─────┘   └───────┬─────┘
        │               │
        └───────┬───────┘
                │
                ▼
┌───────────────────────────────┐
│⑤ 输出：证据列表 + 路径摘要      │
└───────────────┬───────────────┘
                │
                ▼
┌───────────────────────────────┐
│⑥ 生成：问题 + 证据文本 → 答案   │
└───────────────────────────────┘
```

**图注**：**③** 与 **④** 汇合后由 **②** 统一裁决下一步扩展、剪枝与回溯。**③** 提供隐状态与快照；**④** 提供子节点分数或排序。**⑥** 只消费已定稿的 **文本证据**。

**三句边界**：**往哪走、何时退** 由 **②** 决定；**③** 提供沿路可恢复的状态表示；**⑥** 不直接继承 **③** 的内部张量，只读证据文本。

## 5. 从问题到答案：数据与推理流程

以下按 **单条样本** 说明信息流。

1. **输入**：自然语言问题及关联 **树**（节点文本、父子关系、叶子索引等）。当前数据来自 **2WikiMultiHopQA** 子集与对应树构建流程。
2. **控制器驱动导航**：在深度、节点数、证据条数等限制下，循环 **扩展子节点 → 评估路径与证据 → 必要时回溯**。需要排序时调用 **Router**；需要将当前访问节点纳入路径表示时调用 **导航编码器（Mamba 或对照编码器）**。
3. **导航编码器一步**：对当前节点文本编码，更新 **固定维隐状态**；在决策点 **保存快照**，回溯时 **恢复**。输出服务于 **控制与排序相关特征**，**不是**最终答案。默认情况下每步主要编码 **本步被选中的一个节点**；跨步路径摘要依赖 **向量融合与快照栈**，与「把整段路径原文单次送入 SSM」不等价。若采用路径条件化等扩展，须在实验中 **单独说明** 并与默认结果 **分开报告**。
4. **Router 一步**：在候选子节点上，根据 **问题、子节点文本、导航器给出的标量特征** 等输出排序或分数，交回控制器。Router **不直接改写** 导航器内部的 token 序列，但通过 **次序与剪枝** 影响后续节点序列。
5. **形成生成材料**：得到证据列表与路径摘要，再经上下文选择与截断，形成送入生成模型的可读上下文。
6. **生成与评价**：生成模型基于问题与证据输出答案。评价时建议 **过程指标**（金叶是否被访问、是否进入上下文等）与 **答案正确率** **分开讨论**。

## 6. 目前已开展的工作

在 **2WikiMultiHopQA 真实子集** 上，已完成或推进的工作包括：

- **端到端链路**：从树与问题出发，经导航得到证据，再由固定大模型生成答案，全流程可运行。
- **批量与复现**：在统一样本清单上组织导航与可选端到端批量运行，保留批次与样本级记录。
- **过程度量**：接受门与审计类摘要，用于区分「从未走到含金叶」与「已 visit 但未进入可接受证据」等，支撑上述研究问题的归因。
- **策略与旋钮**：在同一树与同一清单下，对比规则排序、余弦类排序、根层轻量学习与规则混合等 Router 变体，并结合实体先验、探索预算等控制器侧旋钮做阶段性收敛；对导航编码器做过 **与默认协议对齐的对照实验**（结论依赖协议表述，详见实验记录）。
- **路径信息的扩展尝试**：实现将 **路径条件化文本** 并入单次前向的扩展配置，并完成 **小规模与中规模（如 50/500 条）** 试验；是否继续扩大规模由 **过程指标与算力** 综合决定。
- **硬样本发现与回溯对比实验（计划）**：为验证快照回溯机制的独立价值，计划从 2WikiMultiHopQA 全集中筛选 **需要回溯的困难样本**（例如纯贪心导航成功率显著低于 SSGS 的子集），并在此基础上做三臂对比：**无回溯** / **固定维快照回溯** / **全量重编码回溯**，重点度量导航成功率恢复幅度与单次回溯效率。

具体配置名、批次号与数字表见 **实验记录**。

## 7. 目前局限性与开放点

- **数据与设置**：目前主要在 **2WikiMultiHopQA** 子集及已声明的导航协议下做系统验证；后续向其他数据域或协议推广，须重新声明与实验。
- **Oracle** 仅作上界参照，不与可部署导航策略混为同一结论层次。
- **路径混合与固定维容量** 须在论文中主动讨论，并以消融支撑。
- **树构建质量** 与 **方法族外部对照**（RQ3）须在成稿中写清假设与规模边界。
- **Router 与导航协议**：当前排序特征以 **文本与标量导航信号** 为主；若未来让 **隐状态向量直接参与排序**，需单独定义特征合同并做对照实验。
- **编码器适配**：若引入 **隐状态向量参与排序**，其与现有 **文本特征** 若不在同一表示空间，可能需要 **投影或联合训练**，并 **同步微调编码器**（含 Mamba 侧）以适配新特征分布，否则难以稳定发挥。
- **结构扩展**：图按查询展开为搜索树、并行搜索提升效率、mamba架构在连续空间利用状态插值获得“模糊回溯”等，在树状主线与结论稳定后再讨论。
- **机制开放点**：Mamba 内部是否存在类似 Transformer 的 **显式检索/注意力式** 机制、能否用于导航中的证据检索和定位，留作后续工作与讨论。

## 8. 构思与后续方向（组内讨论，非结题承诺）

以下吸收近期 **「进攻式叙事」** 头脑风暴及随后的 **批判性整理**：**不把瓶颈说成不存在**，而区分 **改任务契约、改系统拼接、改分析对象、改训练目标** 四条进路；**任一条落地前** 须各自写清 **假设、负对照、与主 RQ 的绑定方式**，避免「换了子任务却写不出 SSM 必要性」。

1. **路径一致性 / 漂移类判别（任务侧）**——若路径信息主要经 **外存文本**（如拼接摘要）进入模型，则须预先回答 **相对轻量分类基线** 的增益域；更有约束力的版本是 **以访问链上的向量序列为输入**、弱化全文外存依赖，再讨论 **序列压缩是否必要**。（标签定义与数据构造成本高。）

2. **粗粒度快照 + 细粒度残差（系统侧）**——在 **承认有损快照** 的前提下，用 **极小局部状态** 补偿恢复质量，可组织 **显存—效果** 类曲线；默认定位 **补充分析或系统章节**，除非与 **树导航主问题** 形成 **不可替代** 的绑定叙述。

3. **增量或差分快照（分析侧）**——对 **选择性 SSM**，**简单线性分解** 往往 **不成立**；在缺乏 **界或消融** 前，**不作为核心方法**，至多作为 **探索性实验** 报告正负结果。

4. **回溯相关的一致性训练目标（训练侧）**——若采用 **不同路径到达同一节点** 的表征约束，须谨慎处理 **「节点语义本就依赖路径」** 的情形，避免正则与任务语义冲突；与 **原有导航 / 语言建模目标** 的张力、系数与 **样本覆盖** 须预先设计。

**倾向备忘（未冻结）**：若主线强调 **「承认损失—用对照与目标缓解」**，训练侧或协议侧更可 **证伪**；若以 **判别探针** 作证据，宜写成 **表征行为分析**，避免 **反客为主**、冲淡 **SSM 在声明契约下的角色定义**。
