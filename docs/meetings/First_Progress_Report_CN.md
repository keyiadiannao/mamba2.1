# 树状 RAG 与多步回溯导航 — 阶段工作说明

面向导师/课题组的简要说明：**研究什么问题、核心创新与研究取向、系统如何工作、目前已做到哪一步**。不涉及具体实验数值与仓库路径；更细的实验台账由课题组内部技术记录维护。

---

## 摘要

长文档问答若仅用扁平检索，难以同时利用 **主题层级** 与 **句级证据**；多步决策又容易 **早走偏支**。本工作研究 **在树状结构上做多步导航与回溯**：用 **Mamba 的固定维隐状态与节点级快照** 支撑「走错可退、退后可续搜」；用 **Router 与 SSGS 控制器** 显式承担 **子节点排序、深度与证据预算、接受与回溯逻辑**。生成侧采用成熟大模型，**只读最终整理后的文本证据**，以便把 **「证据如何获得」** 与 **「答案如何生成」** 在分析上分开。核心科学问题集中在 **回溯控制是否可靠、导航行为能否被度量并与下游证据质量对齐**，而非「检索器与语言模型是否分成两个模块」——后者与常见 RAG 类似，**不是**本课题要单独回答的创新点。

---

## 1. 要解决什么问题

| 方面 | 内容 |
|:---|:---|
| **结构** | 知识以 **树** 组织，兼顾高层概览与底层叶子，比纯扁平切块更贴近长文档与多跳阅读。 |
| **控制** | 多步探索需要 **试探—评估—回溯**；需要明确 **何时扩展、何时回退、何时停止、证据如何进入上下文**。 |
| **状态与代价** | 长链路上若依赖随序列增长的中间状态， **回退与重算** 代价高；希望在导航侧探索 **固定大小状态 + 可恢复快照** 是否有利于 **深层、可回溯** 树上的受控搜索。 |
| **分析需求** | 需要能区分 **「从未走到含金证据的枝」** 与 **「走到了但未进入可用上下文」** 等，避免把生成端噪声误记为导航收益。 |

---

## 2. 核心研究问题（修订表述）

与总框架文档一致，但 **RQ2 不再以「Navigator 与 Generator 是否解耦」为核心**——常见 RAG 亦将检索与生成分开；本课题的 **系统层问题** 在于 **树状多步回溯这一控制形态** 能否在工程上 **稳定运行**，并为 **过程度量与归因** 提供基础。

| 编号 | 问题 |
|:---|:---|
| **RQ1（机制）** | **Mamba 的状态快照与恢复**，能否 **稳定支撑** 树状检索中的 **回溯控制**？ |
| **RQ2（系统/可控）** | 在上述 **树状多步回溯** 框架下，**搜索与证据获取过程** 能否 **可重复、可度量**，并支持 **导航侧行为与证据/上下文质量** 的对照分析（为讨论最终答案质量提供接口）？ |
| **RQ3（价值）** | 在 **深层探索或资源受限** 等场景下，基于 **状态快照** 的导航方式是否在 **系统层面** 具有可讨论的优势（**不**预设等价于「全面榜单碾压」）？ |

---

## 3. 核心创新与研究取向（摘要展开）

**创新重心在「树 + 多步回溯 + 固定维状态载体」这一组合，而非「两个模块分开」本身。**

1. **机制层**：将 **Mamba 固定维隐状态** 用作沿路径读树时的 **状态更新器**，并在决策点使用 **快照保存与恢复**，使 **回溯** 成为可实现的控制原语，而不是仅停留在算法描述。  
2. **控制层**：**SSGS 控制器** 将 **深度优先式探索、预算、接受门、回溯触发** 等与 **Router 的子节点排序** 分离表述——**Router 不充当编码器**，**Mamba 不充当路径决策器**，职责边界清晰。  
3. **取向层**：**先建立可分析、可对照的导航—证据过程**，再审慎讨论 **向最终答案质量的传导**；**不**把「Mamba 替代 Transformer 做生成」或「无归因的榜单跃升」作为现阶段主结论。  
4. **延展（占位）**：若后续数据与实验支持，框架可讨论向 **由查询展开的结构化搜索树** 迁移；当前主线仍以 **真实树上的导航与评测协议** 为主。

## 4. 系统构成与模块关系（纯文字结构图）

下列为 **概念层级**，自上而下阅读即可；不涉及实现细节或程序名称。

```
                    ┌─────────────────────────────────────┐
                    │  输入：用户问题 + 树状文档结构表示      │
                    └──────────────────┬──────────────────┘
                                       ▼
    ┌──────────────────────────────────────────────────────────────┐
    │  搜索控制器（SSGS）                                            │
    │  · 在树上组织多步探索（如深度优先）                              │
    │  · 维护深度、节点数、证据条数等约束                               │
    │  · 决定何时接受证据、何时判定路径不佳、何时回溯                    │
    └───────────────┬──────────────────────────────┬───────────────┘
                    │                              │
         需要「当前路径上的向量表示」              │  需要「子节点谁优先」
                    ▼                              ▼
    ┌───────────────────────────┐    ┌───────────────────────────┐
    │  导航编码器（Mamba 承担）     │    │  路由器（Router）          │
    │  · 顺序读当前节点文本         │    │  · 对候选子节点打分与排序   │
    │  · 更新固定维隐状态           │    │  · 多种策略可替换对比      │
    │  · 快照保存 / 恢复以支持回溯   │    └─────────────┬─────────────┘
    └─────────────┬─────────────┘                    │
                  │                                  │
                  └──────────────┬───────────────────┘
                                 ▼
                    ┌─────────────────────────────────────┐
                    │  输出：证据文本列表 + 可选路径摘要      │
                    │  （供下游整理为生成模型可读上下文）       │
                    └──────────────────┬──────────────────┘
                                       ▼
                    ┌─────────────────────────────────────┐
                    │  生成模型                              │
                    │  · 仅基于「问题 + 证据文本」生成答案      │
                    │  · 不使用导航器内部连续状态              │
                    └─────────────────────────────────────┘
```

**三句边界**：路径 **往哪走、何时退** 由 **控制器与路由器** 决定；**Mamba** 提供 **沿路的可恢复状态表示**；**生成** 只消费 **已定稿的文本证据**。

## 5. 从问题到答案：数据与推理流程

以下按 **单条问答样本** 说明 **信息如何流经各模块**（与是否批量跑实验无关）。

1. **输入**  
   系统获得 **自然语言问题** 及与之关联的 **树**（节点文本、父子关系、叶子与主题的对应关系等）。

2. **搜索控制器驱动多步导航**  
   从根或当前焦点出发，在 **深度、节点数、证据数量** 等限制下，反复进行 **扩展子节点 → 评估当前路径与证据 → 必要时回溯**。  
   需要 **子节点排序** 时，控制器调用 **路由器**；需要 **把当前访问节点纳入可携带的路径表示** 时，调用 **Mamba 导航编码器**。

3. **导航编码器的一步**  
   对 **当前节点文本** 做顺序编码，更新 **固定大小的隐状态**；在预定位置 **保存快照**；若控制器决定回溯，则 **恢复** 到先前快照并改试其他分支。其输出是 **供排序与控制使用的表示**，**不是**最终自然语言答案。

4. **路由器的一步**  
   在候选子节点上，根据 **问题、子节点文本、导航器给出的特征** 等，按所选策略输出 **排序或分数**，交回控制器决定访问次序与剪枝。

5. **形成生成阶段可用的材料**  
   导航结束后，得到 **证据文本列表** 及可选的 **路径摘要**；再经 **上下文选择与截断**（课题中另有独立约定），形成送入生成模型的 **可读上下文**。

6. **生成与评价**  
   生成模型基于 **问题与证据文本** 输出答案。评价时可将 **「金证据是否曾被访问、是否进入上下文」** 等 **过程侧** 与 **答案正确率** 等 **结果侧** **分开讨论**，避免混因。

## 6. 目前已开展的工作（概述，不列数据）

在 **真实公开数据子集** 上，课题组已完成或正在推进的工作大致包括：

- **端到端链路**：从树与问题出发，经导航得到证据，再交由固定大模型生成答案的 **全流程可运行**；  
- **批量与可复现**：在统一样本清单上组织 **导航与可选端到端** 的批量运行，并保留 **批次与样本级记录**，便于复查与对比；  
- **过程度量**：引入 **接受门与审计类摘要**，用于区分 **「从未走到含金叶」** 与 **「已 visit 但未进入可接受证据」** 等，支撑 **RQ2** 所要求的 **可度量、可对照**；  
- **策略探索**：在 **同一树与同一清单** 下，对比 **规则排序、余弦类排序、根层轻量可学习排序与规则混合** 等 **路由器变体**，并结合 **实体先验、探索预算** 等 **控制器侧** 旋钮做 **单变量** 与 **阶段性默认工作点** 收敛；  
- **叙事与判停规范**：约定 **过程指标与答案指标并列**、**无效批次不参与优劣比较** 等，避免过度解读单次波动。

具体 **配置、批次标识与数字表格** 属于课题组内部实验记录，不在此文中展开；需要量化材料时再另行提供。

## 7. 主张边界（建议口头一并交代）

- **不主张**：Mamba **全面优于** Transformer；Mamba **取代** 大模型做 **最终生成**；在无严格归因下宣称 **榜单式全面领先**。  
- **上界参照**：**Oracle 式「直接给定证据」** 仅用于 **差距参照**，**不与**真实导航策略混为同一结论层次。
